# SiMDel

**Si**mple **m**olecular **d**ynamics **e**xpandable **l**ibrary - простая библиотека для молекулярной динамики.

### Введение

Позволяет выполнять подготовку и расчет молекулярной динамики напрямую из python. Не требует редактирования файлов или конвертации в другие форматы,
полностью бесшовная молекулярная динамика!

**Главные отличия от других библиотек**:

- Все типизировано, и все стадии всегда представляют собой функции с понятной сигнатурой
- Простая и быстрая агрегация информации о молекулярной системе в виде `pandas.DataFrame`
- Модульная структура библиотеки - просто добавлять новые функции, МД движки или сложные конвейеры
- Универсальный шаблон для конвейера вычислений - пайплайн

### Установка

Библиотека требует несколько типов зависимостей: основные, conda, GROMACS, PLUMED. Можно установить не все, тогда часть библиотеки будет неактивна.

Допускаются другие варианты и комбинации установки pypi+conda зависимостей. Главное - сохранять версии pypi и conda пакетов одинаковыми, чтобы избежать перезаписи пакетов их двойниками с дугой версией

**Порядок установки нельзя менять!**

#### 1. GROMACS

_Обязательно_

Варианты:

1.  **Обычная без** PLUMED ([см. документацию](https://manual.gromacs.org/documentation/current/install-guide/index.html))
2.  **Обычная c** PLUMED ([PLUMED флаг](https://manual.gromacs.org/documentation/current/install-guide/index.html#building-with-plumed-support))
3.  **Через conda** - нет поддержки GPU, невозможно подключить PLUMED:

    _**(Только после создания среды conda)**_

    ```
    conda install gmx
    ```

#### 2. PLUMED

_Нужен для: метадинамики_

Современный GROMACS имеет флаг сборки с поддержкой PLUMED. Сбока из исходников - [см. документацию](https://www.plumed.org/doc-v2.9/user-doc/html/_installation.html)

#### 3. Сonda зависимости

_Нужен для: openff параметризации, построения графа переходов для FEP_

Установить и активировать среду. После создания среды - можно установить GROMACS **(см выше)**

```
conda env create -n simdel_env python=3.10
conda activate simdel_env
conda install -f conda.yml -y
```

#### 4. Pip зависимости

_Обязательно_

Варианты:

1. **UV** - мягкая установка, чтобы не перезаписать файлы conda

   ```
   uv venv simdel_env
   uv sync --inexact
   ```

2. **Poetry**

   ```
   poetry install
   ```

### 5. PMX пакет

_Нужен для: FEP_

Он устарел, и единственный путь - установить пакет pmx из этого проекта _(скоро обновится)_

```
cd pmx
pip install .
```

## Принцип работы

В основе лежат структура молекулярной системы GROMACS. Это реляционная база данных, и благодаря pandas, можно быстро проводить сложные манипуляции с системами и вычислять любые параметры. Система неизменяема. Вместе с другими основными классами опеспечивает консистентность библиотеке.

Все манипуляции с системой - функции, которые возвращают новую систему как результат работы. Единственный побочный результат - создание папки для промежуточных файлов.

Отдельные функции комбинируются в стадии конвейеров - pipelines, эти стадии также имеют лишь один побочный эффект - создание рабочих папок.

## P.S.

_Самая простая механика видеоигр - стрельба и передвижение на wasd._

Многие библиотеки заточены под выполнение конкретной задачи: параметризации, МД симуляции, анализа, сборки молекулярной системы из отдельных молекул. Таким образом приходится собирать собственный конвейер под себя. Для исследователя это техническая задача, которая должна по минимуму соответствовать требованиям. Это приводит к потере универсальности, масштабируемости и проблемам при адаптации разных инструментов.

Главная из таких проблем - неконсистенстность, разный интерфейс и принцип работы у библиотек. Приходится каждый раз строить конвертеры и помнить об особенностях инструмента. При конвертации форматов могут теряться данные, построение оберток отнимает время, а использование уже готовых фреймворков не позволяет добавлять свои функции без глубокого понимания устройства фреймворков.

Поэтому возникла идея создать максимально простую библиотеку, которая свяжет единым интерфейсом отдельные инструменты. А нет ничего более простого чем вызов функции для аргументов! Это позволит просто добавлять новые функции и типы аргументов в общий котел. Так и появилась `SiMDel`
