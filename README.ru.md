# SiMDel

**Si**mple **m**olecular **d**ynamics **e**xpandable **l**ibrary - простая библиотека для молекулярной динамики.

Позволяет выполнять подготовку и расчет молекулярной динамики напрямую из Python. Не требует редактирования файлов или конвертации в другие форматы - полностью бесшовная молекулярная динамика!

### Основные принципы

**Архитектура:** В основе лежит структура молекулярной системы GROMACS. Это реляционная база данных, поэтому благодаря `pandas` можно быстро проводить сложные манипуляции с системами и вычислять любые параметры.

**Иммутабельность:** Система неизменяема. Вместе с другими основными классами она обеспечивает консистентность всей библиотеки.

**Чистые функции:** Все манипуляции с системой — функции, которые возвращают новую систему как результат работы. Единственный побочный результат - создание папки для промежуточных файлов. Простой запуск как локально, так и удаленно.

**Пайплайны:** Отдельные функции комбинируются в стадии конвейеров (pipelines). Эти стадии также имеют лишь один побочный эффект — создание рабочих директорий.

### Установка

Библиотека требует нескольких типов зависимостей: основные, conda, GROMACS, PLUMED. Можно установить не все, тогда часть библиотеки будет неактивна.

Допускаются другие варианты и комбинации установки pypi+conda зависимостей.

> [!IMPORTANT]
> **1. Порядок установки нельзя менять!**
>
> **2. Нужно сохранять версии pypi и conda пакетов одинаковыми**. Это позволит избежать перезаписи пакетов их двойниками с другой версией

#### 1. GROMACS

_Обязательно_

Варианты:

1.  **Обычная без** PLUMED ([см. документацию](https://manual.gromacs.org/documentation/current/install-guide/index.html))
2.  **Обычная c** PLUMED ([PLUMED флаг](https://manual.gromacs.org/documentation/current/install-guide/index.html#building-with-plumed-support))
3.  **Через conda** - нет поддержки GPU, невозможно подключить PLUMED: **см ниже**

#### 2. PLUMED

_**Применение**: метадинамика_

Нужно собрать GROMACS под использование PLUMED! Сборка из исходников - [см. документацию](https://www.plumed.org/doc-v2.9/user-doc/html/_installation.html)

#### 3. Сonda зависимости

_**Применение**: openff параметризация, построение графа переходов для FEP_

Работает только на Linux/WSL/Mac.

> ```bash
> conda env create -n simdel_env python=3.10
> conda activate simdel_env
> conda install -f conda.yml -y
> ```

> [!NOTE] Только после создания среды conda можно установить GROMACS из conda
>
> ```bash
>  conda install gmx
> ```

#### 4. Pip зависимости

_Обязательно_

Варианты:

1. **UV** - мягкая установка, чтобы не перезаписать файлы conda

   ```bash
   uv venv simdel_env
   uv sync --inexact
   ```

2. **Poetry**

   ```bash
   poetry install
   ```

### 5. PMX пакет

_Нужен для: FEP_

> [!WARNING]
> Оригинальный пакет устарел. Единственный рабочий путь сейчас — установить форк pmx из этого проекта _(скоро обновится)_:

```bash
cd pmx
pip install .
```

## P.S.

_Самая простая механика видеоигр - стрельба и передвижение на wasd._

Многие библиотеки заточены под выполнение конкретной задачи: параметризации, МД симуляции, анализа, сборки молекулярной системы из отдельных молекул. Таким образом приходится собирать собственный конвейер под себя. Это приводит к потере универсальности, масштабируемости и проблемам при адаптации разных инструментов.

Главная из таких проблем - неконсистенстность, разный интерфейс и принцип работы у библиотек. Приходится помнить об проблемах инструмента, особенностях конвертации форматов, поэтому построение оберток отнимает много времени. Использование готовых фреймворков не позволяет добавлять свои функции без глубокого внедрения внутрь кода.

Поэтому возникла идея создать максимально простую библиотеку, которая свяжет единым интерфейсом отдельные инструменты. Нет ничего более простого чем вызов функции для аргументов! Это позволит просто дописывать новые функции и типы аргументов в общий котел. Так и появилась `SiMDel`
