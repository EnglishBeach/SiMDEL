# SiMDel

[Версия на анлийском](README.md)

**Si**mple **m**olecular **d**ynamics **e**xpandable **l**ibrary - простая библиотека для молекулярной динамики.

Позволяет выполнять подготовку и расчет молекулярной динамики напрямую из Python. Не требует редактирования файлов или конвертации в другие форматы - полностью бесшовная молекулярная динамика!

### Основные принципы

**Архитектура:** В основе лежит структура молекулярной системы GROMACS. Это реляционная база данных, поэтому благодаря `pandas` можно быстро проводить сложные манипуляции с системами.

**Иммутабельность:** Система неизменяема. Вместе с другими основными классами она обеспечивает консистентность всей библиотеки.

**Чистые функции:** Все манипуляции с системой — функции, которые возвращают новую систему как результат работы. Единственный побочный результат - создание папки для промежуточных файлов. Простой запуск как локально, так и удаленно.

**Пайплайны:** Отдельные функции комбинируются в стадии конвейеров (pipelines). Эти стадии также имеют лишь один побочный эффект — создание рабочих директорий.

### Установка

Библиотека требует нескольких типов зависимостей: PyPi, conda, GROMACS, PLUMED. Можно установить не все, тогда часть библиотеки будет неактивна.

> [!IMPORTANT]
>
> 1. Допускаются другие варианты и комбинации установки PyPi+conda зависимостей.
> 2. Порядок установки нельзя менять!
> 3. Нужно сохранять версии PyPi и conda пакетов одинаковыми. Это позволит избежать перезаписи пакетов их двойниками с другой версией

#### 1. GROMACS

_Обязательно_

Варианты:

1.  **Обычная без** PLUMED ([см. документацию](https://manual.gromacs.org/documentation/current/install-guide/index.html))
2.  **Обычная c** PLUMED ([PLUMED флаг](https://manual.gromacs.org/documentation/current/install-guide/index.html#building-with-plumed-support))
3.  **Через conda** - нет поддержки GPU, невозможно подключить PLUMED: **см ниже**

#### 2. PLUMED

_**Применение**: метадинамика_

Нужно собрать GROMACS под использование PLUMED! Сборка из исходников - [см. документацию](https://www.plumed.org/doc-v2.9/user-doc/html/_installation.html)

#### 3. Сonda зависимости

_**Применение**: openff параметризация, построение графа переходов для FEP_

Работает только на Linux/WSL/Mac.

> ```bash
> conda create -n simdel_env python=3.10
> conda activate simdel_env
> conda install -f conda.yml -y
> ```

> [!NOTE] При установке GROMACS через conda: сначала установить среду, а потом GROMACS
>
> ```bash
>  conda install gromacs
> ```

#### 4. Pip зависимости

_Обязательно_

Варианты:

1. **UV**

```bash
uv pip install .
```

2. **Poetry**

```bash
poetry install
```

### 5. PMX пакет

_**Применение**: FEP_

> [!WARNING]
> Оригинальный пакет устарел. Единственный рабочий путь сейчас — установить форк pmx из этого проекта

```bash
cd pmx
pip install .
```

## P.S.

_Самая простая механика видеоигр - стрельба и передвижение на WASD._

Многие библиотеки заточены под выполнение конкретной задачи: параметризации, МД симуляции, анализа, сборки молекулярной системы из отдельных молекул. Таким образом приходится собирать конвейер под себя. Это приводит к потере универсальности, масштабируемости и проблемам при адаптации разных инструментов.

Главная из таких проблем - разный интерфейс и принцип работы у библиотек. Приходится помнить об проблемах инструмента, особенностях конвертации форматов, поэтому построение оберток отнимает много времени. Использование готовых фреймворков не позволяет добавлять свои функции без глубокого внедрения внутрь кода.

Поэтому возникла идея: создать максимально простую библиотеку, которая свяжет единым интерфейсом отдельные инструменты. Что модет быть проще вызова функции от аргументов? Такой подход позволяет без труда дописывать новые функции и типы аргументов в общий котел. Так и появилась `SiMDel`
